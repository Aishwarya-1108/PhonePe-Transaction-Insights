PhonePe Transaction Insights & Customer Segmentation

🚀 Overview

This project analyzes PhonePe digital payment data to extract meaningful insights on transactions, user engagement, and regional performance. 

🎯 Objectives
Analyze transaction trends across states
Identify high and low performing regions
Understand user engagement patterns
Perform customer segmentation using clustering

🛠️ Tech Stack
Language: Python
Database: PostgreSQL
Libraries: Pandas, Scikit-learn
Visualization: Streamlit, Plotly

📂 Dataset
Source: PhonePe Pulse
Includes:
Aggregated Transactions
User Data
Insurance Data

 Key Features
State-wise transaction analysis
Year-wise trend analysis
User and insurance insights
Customer segmentation using KMeans
Interactive Streamlit dashboard

📊 Dashboard
Interactive filters (state, year, quarter)
Dynamic charts and graphs
Cluster visualization

▶️ How to Run
git clone https://github.com/your-username/phonepe-analysis.git
cd phonepe-analysis
pip install -r requirements.txt
streamlit run app.py

📈 Key Insights
Few states contribute to most transaction value
High transaction count ≠ high transaction value
Identified regions with growth potential

Conclusion
This project highlights how data analytics and machine learning can be used to understand digital payment trends and support business decisions.
